### 기기에 현금 투입

컬러 센서의 인식 값은 프로그램이 시작될 때 임의의 값으로 초기화 됨
1. 	화면에서 건물 구입 금액이 결정됨
2. 	현재 플레이어가 그 금액 이상을 가지고 있는지 확인
3. 	그만큼의 금액이 경우 처음 화면으로 돌아감
4. 	Pi에서 AVR로 신호 전송(입력 모터 동작 신호 + 가격)


1. 	모터가 돌아가기 시작 (포토인터럽트 사용 X)
2. 	컬러 센서가 읽는 값이 바뀌었을 경우, 이를 새로운 지폐로 인식
2. 	지폐가 새로 입력된 것이 확인되면, 모터 정지
3. 	원판의 해당 지폐 수납 칸을 지폐가 떨어질 레일에 맞춤
4. 	레일의 모터가 천천히 돌아가면서, 지폐를 원판으로 떨어트림.
3. 	지폐가 떨어지기까지는 일정 시간이 걸린다고 가정하고 프로그램 작성 (예 : 5초)
4. 	그 시간이 흐르면, 다시 투입구 모터가 돌아가기 시작
5. 	위의 작업을 반복한 후, 금액이 초과되면 입구 모터를 돌리지 않고, 거스름돈을 반환함.
6. 	모든 작업이 끝나면, AVR은 Pi에게 완료 신호를 보냄

컬러 센서의 이전 값을 계속 저장하고 있는 변수가 필요. 프로그램이 시작되자마자 읽는 컬러 센서 값이 지폐가 없을 때 인식하는 값이므로, 그것을 Default 값으로 저장해놓고 사용.

파이에서 AVR로 넘어오는 데이터는 두 가지. 모터 동작 신호와 가격임. AVR이 가격을 알고 있으면, 혼자서 그 가격의 입력을 처리하고 다시 파이에게 통보할 수 있음. 파이는 동작 신호를 전송한 후 AVR이 무슨 신호를 돌려줄 때 까지 대기
